# เอกสารสรุปรูปแบบและเทคโนโลยีของระบบ (Project Overview)

เอกสารนี้จัดทำขึ้นสำหรับผู้บริหารและคณะทำงาน เพื่อให้เห็นภาพรวมว่า "ทำไม" เราถึงเลือกใช้รูปแบบและเทคโนโลยีชุดนี้ในการพัฒนา และ "ประโยชน์" ที่หน่วยงานจะได้รับคืออะไร โดยจะละเว้นรายละเอียดเชิงเทคนิคที่ซับซ้อน (ซึ่งฝ่ายเทคนิค/ผู้ดูแลระบบจะเป็นผู้ดำเนินการ)

---

## 1. แนวคิดหลัก (Core Concepts)
เราพัฒนาระบบนี้โดยยึดหลัก **"เร็ว, รองรับอนาคต, และดูแลง่าย"** เพื่อให้เกิดความคุ้มค่าสูงสุดในระยะยาว

---

## 2. ทำไมต้องใช้ Stack นี้? (Technology Benefits)


### 2.1 หน้าบ้าน (Frontend) : Next.js 16 + Tailwind CSS
**มันคืออะไร:** ส่วนที่แสดงผลให้ผู้ใช้งานเห็น (หน้าเว็บ ปุ่ม รูปภาพ)
**ทำไมถึงเลือกใช้:**
- **รวดเร็ว (High Performance):** เน้นการโหลดหน้าที่ไวมาก ผู้ใช้งานไม่ต้องรอนาน ลดโอกาสที่คนจะปิดเว็บหนี
- **ภาพลักษณ์ทันสมัย (Modern Design):** Tailwind ช่วยให้เราปรับแต่งหน้าตาเว็บให้ดูดี เป็นสากล และแสดงผลได้สมบูรณ์แบบทั้งบนคอมและมือถือ (Responsive)
- **หาคนดูแลง่าย:** เป็นเทคโนโลยีที่นักพัฒนาส่วนใหญ่ในตลาดใช้กันเป็นมาตรฐาน ทำให้ในอนาคตถ้าเปลี่ยนทีมพัฒนา ก็สามารถหาคนมาทำต่อได้ทันที

### 2.2 คลังข้อมูล (Database) : PostgreSQL
**มันคืออะไร:** ตู้เซฟสำหรับเก็บข้อมูลสำคัญทั้งหมดขององค์กร (เช่น รายชื่อสมาชิก, ไฟล์งานวิจัย, ข่าวสาร)
**ทำไมถึงเลือกใช้:**
- **ความน่าเชื่อถือสูงสุด:** เป็นระบบฐานข้อมูลระดับ Enterprise ที่ธนาคารและองค์กรใหญ่ทั่วโลกเลือกใช้ มั่นใจได้ว่าข้อมูลจะไม่สูญหายหรือเสียหาย
- **รองรับการเติบโต:** ไม่ว่าข้อมูลงานวิจัยจะเพิ่มเป็นหมื่นหรือแสนชิ้น ระบบก็ยังทำงานได้เสถียร ไม่ช้าลง

### 2.3 "ล่ามแปลภาษา" ข้อมูล (ORM) : Prisma
**มันคืออะไร:** เครื่องมือที่ช่วยให้โปรแกรมเว็บไซต์คุยกับฐานข้อมูลรู้เรื่อง
**ทำไมถึงเลือกใช้:**
- **ความแม่นยำ:** ป้องกันความผิดพลาดในการบันทึกข้อมูล เช่น กรอกวันที่ผิดฟอร์ม หรือลืมใส่ข้อมูลสำคัญ ระบบจะแจ้งเตือนทันที
- **ดูแลรักษาง่าย:** หากมีการเปลี่ยนรูปแบบการเก็บข้อมูลในอนาคต เราสามารถแก้ไขจุดเดียวแล้วให้ระบบจัดการความเปลี่ยนแปลงทั้งหมดให้เอง ลดเวลาการทำงานของทีมพัฒนาลงได้มาก

### 2.4 ระบบค้นหาอัจฉริยะ : Meilisearch
**มันคืออะไร:** ระบบค้นหาที่ทำงานแยกออกมาเป็นพิเศษ เหมือน Google ส่วนตัวของเว็บไซต์
**ทำไมถึงเลือกใช้:**
- **ประสบการณ์ผู้ใช้ที่ดีเยี่ยม:** การค้นหาแบบปกติมักจะช้าและต้องพิมพ์ถูกทุกตัวอักษร แต่ Meilisearch ช่วยให้ผู้ใช้พิมพ์แค่บางคำ ระบบก็เดาและแสดงผลลัพธ์ขึ้นมาให้ทันที (Instant Search) ช่วยให้คนหางานวิจัยเจอได้ง่ายและเร็วขึ้นมาก

---

## 3. ฟีเจอร์พิเศษ (Special Feature) : ระบบอัตโนมัติ n8n
**"พนักงานเสมือนที่ทำงานแทนคุณ 24 ชม."**

นี่คือ "ไม้ตาย" ของระบบนี้ที่ช่วยลดภาระงานของเจ้าหน้าที่ได้อย่างมหาศาล
**ทำไมมันถึงพิเศษ?**
- **คุยได้ทุกภาษา:** n8n สามารถเชื่อมต่อระบบงานวิจัยของเรา เข้ากับ **LINE Official, Email, Google Sheets** หรือแม้แต่ระบบฐานข้อมูลเดิมของโรงพยาบาล
- **ทำงานแทนคน:**
  - *ตัวอย่าง 1:* เมื่อมีคนส่งงานวิจัยใหม่เข้ามา -> ให้ส่ง LINE แจ้งเตือนกรรมการตรวจทานทันที
  - *ตัวอย่าง 2:* ทุกสิ้นเดือน -> ให้ดึงสถิติคนเข้าเว็บใส่ Excel แล้วส่งเมลหาผู้บริหาร
- **ปรับเปลี่ยนได้เอง:** รูปแบบการทำงาน (Workflow) เป็นแบบลากวาง (Drag & Drop) ผู้ดูแลสามารถปรับเปลี่ยนเงื่อนไขได้เองโดยไม่ต้องจ้างโปรแกรมเมอร์แก้โค้ด

---

## 4. ทำไมต้องใช้ "Docker"? (Infrastructure)

**มันคืออะไร:** เทคโนโลยีการจำลองสภาพแวดล้อมของ Server (เปรียบเหมือน "กล่องสำเร็จรูป" ที่แพ็คระบบทั้งหมดไว้)
**ทำไมถึงเลือกใช้ (ประโยชน์ต่อองค์กร):**
1. **ลดปัญหาระบบล่ม:** เมื่อย้ายระบบจากเครื่องทดสอบไปเครื่องจริง จะไม่มีปัญหา "เครื่องฉันรันได้ เครื่องเธอรันไม่ได้" เพราะทุกอย่างถูกแพ็คไปเหมือนกัน 100%
2. **ย้ายเครื่องง่าย:** หากวันหนึ่ง Server ปัจจุบันเสีย หรือต้องการย้าย Cloud ไปเข้าใหม่ เราสามารถยก "กล่อง" นี้ไปวางที่ไหนก็ได้และรันได้ทันทีภายในไม่กี่นาที
3. **ปลอดภัยกว่า:** ถ้ามีไวรัสหรือผู้บุกรุกโจมตี "กล่อง" ใดกล่องหนึ่ง จะไม่ลามไปยังส่วนอื่นของ Server

## 5. โครงสร้างและการจัดการข้อมูล (Data Management)

เรารู้ว่า "ข้อมูล" คือหัวใจ เราจึงออกแบบให้หน้าบ้านและหลังบ้านแยกกันแต่ทำงานร่วมกันได้อย่างราบรื่น

- **ตู้เซฟ (Database):** เก็บแต่ข้อมูลดิบ ไม่ยุ่งกับความสวยงาม
- **พนักงานต้อนรับ (API/Backend):** คอยรับคำสั่งและดึงข้อมูลมาเสิร์ฟ
- **หน้าตกแต่ง (Frontend):** รับข้อมูลมาจัดวางให้สวยงาม

**ข้อดีของการทำแบบนี้:**
- หากต้องการเปลี่ยนหน้าตาเว็บไซต์ใหม่ทั้งหมดในอนาคต **ข้อมูลจะไม่หาย** และไม่ต้องรื้อระบบหลังบ้านใหม่ ทำแค่เปลี่ยน "หน้ากาก" ใหม่ได้เลย ประหยัดงบประมาณในการปรับปรุงในระยะยาว

---

## 6. การนำไปใช้งานจริง (Deployment Strategy)

ทางทีมพัฒนาแนะนำให้ติดตั้งระบบบน **Linux Server**
**เหตุผล:**
- **ประหยัดงบประมาณ:** ระบบ Linux กินทรัพยากรเครื่องน้อยกว่า Windows มาก ทำให้เราไม่ต้องซื้อเครื่อง Server แพงๆ ก็รองรับคนใช้งานได้เยอะเท่ากัน
- **ความเสถียรและความปลอดภัย:** Linux ถูกออกแบบมาเพื่อเปิดใช้งานตลอด 24 ชั่วโมง 365 วัน โดยไม่ต้อง Restart บ่อยๆ ซึ่งเหมาะกับการเป็น Web Server สาธารณะ

*(ส่วนขั้นตอนทางเทคนิคและคำสั่งต่างๆ ทางทีม Dev จะเป็นผู้ดำเนินการเอง)*
